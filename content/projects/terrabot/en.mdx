## Context

A project aimed at simplifying cloud infrastructure provisioning through a
**conversational agent**. The idea: replace the manual configuration of Terraform files
with a natural language dialogue, while providing **comparative cost estimates**
across AWS, GCP and Azure before any deployment.

## Problem

Creating cloud infrastructure with Terraform requires deep technical expertise:
knowledge of providers, available resources, pricing constraints and architectural
best practices. This high barrier to entry slows down teams that want to deploy
quickly without a dedicated DevOps specialist.

## Solution

An LLM agent orchestrated by **LangGraph** that guides the user through a structured
conversation, extracts architecture requirements, compares costs across providers and generates
the corresponding `.tf` file:

- **Requirements gathering**: the agent asks the right questions to extract the target provider,
  region, required services, budget and availability constraints.
- **Cost estimation**: queries to the **Infracost GraphQL API** to price the candidate architecture
  in real time across the three major clouds.
- **Argued proposal**: a summary of the most suitable design with justification for the provider
  choice and a monthly cost projection.
- **Terraform generation**: production of a `.tf` file validated by `terraform validate`,
  triggered only after explicit user confirmation.
- **Live streaming**: LLM tokens are streamed directly into the Streamlit interface
  for a smooth and responsive experience.

## Agent Pipeline

The agent progresses through five explicit nodes with conditional routing:

- **`gather_requirements`** - multi-turn dialogue to extract requirements (provider, region,
  services, budget, SLA).
- **`fetch_pricing`** - Infracost queries to price candidate architectures on AWS,
  GCP and Azure.
- **`propose_architecture`** - synthesis of the optimal design with a cost comparison and
  justification.
- **`await_confirmation`** - explicit validation checkpoint before any code generation.
- **`generate_terraform`** - production and validation of the final `.tf` file.

## Notable Technical Points

- **Infracost mock mode**: a realistic mock client enables development and demos
  without an API key, toggled via an environment variable (`USE_MOCK_INFRACOST=true`).
- **Typed GraphState**: the conversation state is modelled with strict Pydantic types
  (`GraphState`, `ArchitectureRequirements`), ensuring consistency across nodes.
- **Decoupled prompts**: `SYSTEM_PROMPT`, `EXTRACTION_PROMPT` and `PROPOSAL_PROMPT` are
  isolated in `prompts.py` to facilitate iteration without touching the business logic.
